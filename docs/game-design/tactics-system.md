# 戦術システム設計方針

## 基本方針

弟（主人公）の戦い方を**ゲームメカニクスとして機能させる**。

正面戦闘を避け、間接的手段で勇者を追い詰める。

---

## 弟が使う戦術一覧

| 戦術 | 概要 |
|---|---|
| 洗脳 | NPCや組織メンバーを操作する |
| 心理操作 | 勇者パーティの信頼関係を崩す |
| スパイ | 勇者側の情報を収集・利用する |
| 内部崩壊 | パーティ内の分断を誘発する |
| 暗殺 | 孤立させた標的を確実に仕留める |
| 罠 | 行動パターンを読んで先手を打つ |
| 情報戦 | 偽情報の流布・情報遮断 |

---

## 設計上の重要方針

**努力によって勇者を追い詰めることを徹底する。**

- チートで強くなるのではない
- 積み上げた研究・計画の結果として強くなる
- 途中で勇者を本当に追い詰める場面を作る（師匠介入前）

---

## 巨大魔法陣

ラスボス戦の核となるメカニクス。

- 数年かけて構築した集大成
- 勇者パーティを完全に詰める設計
- 師匠の「結界剣」一つで完全無効化される

この**非対称性**がテーマの体験として機能する。
